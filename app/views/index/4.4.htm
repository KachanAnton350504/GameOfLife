4.4 Средства организации баз данных

 В современных системах автоматизации  используются СУБД архитектуры клиент/сервер: на центральном сервере размещается система управления базой данных (СУБД), а на остальных компьютерах сети устанавливается СУБД-клиент, имеющий графический интерфейс для доступа к базе данных. При помощи данных клиентов можно также отправлять запросы для выполнения различных операций СУБД, например, изменение, обновление данных.

         База данных  создана для хранения информации об отделе, его деятельности, финансовом состоянии, данные прошлых и текущих проектов, информация о текущей деятельности, приоритетах.

        В отделе целесообразно использовать распределённую базу данных с размещением баз данных на компьютерах сети, при управлении через программу Microsoft Access.

        К СУБД выдвигаются следующие требования:
необходимо быстрее и более обоснованно принимать деловые решения, увеличивать производительность и гибкость коллектива разработчиков, в условиях снижения совокупного бюджета на информационные технологии продолжать наращивание инфраструктуры в соответствии с постоянно возрастающими потребностями.

Microsoft SQL Server 2014

При автоматизации предприятия будет использоваться СУБД Microsoft SQL Server 2014.

  SQL Server 2014 предоставляет для приложений обработки и анализа данных предприятия повышенную защищенность, масштабируемость и доступность, облегчая их создание, развертывание и обслуживание.

Расширяя возможности SQL Server 2012, SQL Server 2014 обеспечивает интегрированную обработку и анализ данных, что позволяет предприятиям любого масштаба выполнять следующие задачи:

• Построение, развертывание и управление приложениями предприятия с повышением их защищенности, масштабируемости и надежности.

• Максимальное увеличение производительности труда IT-персонала путем уменьшения сложности разработки и поддержки приложений баз данных.

• Совместное использование данных различными платформами, приложениями и устройствами, что облегчает объединение внутренних и внешних систем.

• Ограничение затрат без снижения производительности, доступности, масштабируемости и безопасности.

SQL Server 2014 обеспечивает развитие инфраструктуры данных предприятия в трех ключевых направлениях: управление данными предприятия, производительность разработки и интеллектуальные ресурсы предприятия (Business Intelligence, BI). Помимо этого, он включает в себя инновации в области ценовой политики и лицензирования, а также перехода на SQL Server 2014 и Microsoft Windows Server System.

SQL Server включает в себя следующие компоненты:

Реляционная база данных. Защищенное, надежное и масштабируемое ядро СУБД с высокой доступностью, улучшенной производительностью и поддержкой структурированных и неструктурированных (XML) данных. 

Службы Replication Services. Репликация данных для приложений распределенной и мобильной обработки данных, обеспечения высокой готовности систем, масштабируемого параллелизма с использованием вторичных хранилищ данных для корпоративных решений подготовки отчетов.

Службы Notification Services. Развитые возможности по рассылке уведомлений, предназначенные для целей разработки и развертывания масштабирумых приложений, позволяющие обеспечивать персонифицированное, своевременное обновление информации для большого числа подключенных и мобильных устройств. 

Службы Integration Services. Функции извлечения, трансформации и загрузки данных (ETL) для хранилищ данных и интеграции данных в масштабе предприятия. 

Службы Analysis Services. Поддержка оперативной аналитической обработки (OLAP) для быстрого и глубокого анализа больших и сложных наборов данных с использованием многомерных хранилищ. 

Службы Reporting Services. Комплексное решение для создания, управления и доставки как традиционных бумажных, так и интерактивных веб-отчетов. 

Средства управления. SQL Server включает в себя интегрированные средства для расширенного управления базой данных и ее настройки, а также для тесной интеграции со такими средствами, как Microsoft Operations Manager (MOM) и Microsoft Systems Management Server (SMS). Стандартные протоколы доступа значительно снижают время интеграции данных, находящихся в SQL Server, с существующими системами. Кроме того, в SQL Server встроена поддержка веб-служб, упрощающая взаимодействие с другими приложениями и платформами. 

Средства разработки. SQL Server обеспечивает интегрированные средства разработки для ядра базы данных, извлечения, трансформации и загрузки данных, интеллектуального анализа, OLAP и построения отчетов, которые тесно интегрированы с Microsoft Visual Studio, что позволяет поддерживать полный цикл разработки приложений. Каждая из основных подсистем SQL Server имеет собственную объектную модель и набор интерфейсов прикладных программ (API), позволяющих производить расширение системы данных в необходимом предприятию направлении.

 

Microsoft Access

Кроме того, на компьютеры всех отделов устанавливается программа Microsoft Access, входящая в пакет Microsoft Office 2010.

Microsoft Access - реляционная СУБД от корпорации Microsoft, обладает интуитивно понятным интерфейсом. Имеет широкий спектр функций, включая связанные запросы, сортировку по разным полям, связь с внешними таблицами и базами данных. Благодаря встроенному языку VBA, в самом Access можно писать приложения, работающие с базами данных.

С точки зрения бизнеса программа предоставляет следующие возможности:

• Включение данных из различных источников. Access 2010 поддерживает различные форматы данных, в том числе XML, OLE, ODBC и формат служб Microsoft Windows SharePoint Services.

• Связи между бизнес-системами. Можно связать таблицы таким образом, чтобы одновременно получать доступ к данным из различных баз, работая с формами, отчетами и страницами доступа к данным в Access 2010. Кроме того, можно связывать таблицы из других баз данных Access, электронных таблиц Microsoft Excel, источников данных ODBC, баз данных Microsoft SQL Server и других источников.

• Максимально эффективное использование корпоративных данных.
